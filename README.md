## 📂 Repository Structure

The repository is organized by topics and daily practice challenges to ensure a structured learning curve:

| Folder / Concept | Description |
| --- | --- |
| **`day1` to `day3**` | Basics of C++, syntax foundation, and initial logic building. |
| **`day2/ if-else`** | Conditional branching and decision-making structures. |
| **`day4/ pattern`** | Nested loops practice through various coordinate and shape pattern programs. |
| **`day5` to `day7**` | Intermediate logic, mathematical problems, and array fundamentals. |
| **`arrayfully`** | Deep dive into 1D/2D arrays, manipulation, and optimization. |
| **`pointer` / `pointers` / `pointer fully**` | Memory addresses, reference variables, and mastering pointer arithmetic. |
| **`refrence`** | Understanding pass-by-reference vs. pass-by-value in functions. |
| **`dynamic` / `dynamic memory**` | Heap allocation using `new` and `delete` operators to prevent memory leaks. |
| **`oops`** | Object-Oriented Programming principles: Classes, Objects, Inheritance, Polymorphism, Encapsulation, and Abstraction. |
| **`c++ with promblems`** | Dedicated practice solving competitive programming and algorithmic data structure challenges. |

## 🏁 How to Run Local Files

To compile and run any of the C++ programs locally, use a standard GCC/G++ compiler:

```bash
# Compile the file
g++ filename.cpp -o output

# Run the executable
./output

```
